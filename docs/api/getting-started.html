Getting Started with Pitchey API

Welcome to the Pitchey API! This comprehensive guide will help you get up and running with our API to build powerful integrations for the movie industry platform.

Authentication

The Pitchey API uses JWT (JSON Web Tokens) for authentication. You'll need to obtain a token by logging in through one of our portal-specific endpoints.

Portal-Specific Login Endpoints

User Type Endpoint Description
Creator POST /api/auth/creator/login For filmmakers and content creators
Investor POST /api/auth/investor/login For investors and funding organizations
Production POST /api/auth/production/login For production companies and studios
Universal POST /api/auth/login Auto-detects user type

Getting Your First Token

Here's how to authenticate and get your access token:

// Example: Login as a creator
const loginResponse = await fetch('https://pitchey-api-prod.ndlovucavelle.workers.dev/api/auth/creator/login', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    email: 'your-email@example.com',
    password: 'your-password'
  })
});

const loginData = await loginResponse.json();
const token = loginData.token;

// Store the token for future requests
localStorage.setItem('pitchey_token', token);

Using the Token

Include your JWT token in the Authorization header for all authenticated requests:

// Making authenticated requests
const response = await fetch('https://pitchey-api-prod.ndlovucavelle.workers.dev/api/user/profile', {
  headers: {
    'Authorization': `Bearer ${token}`,
    'Content-Type': 'application/json'
  }
});

const userProfile = await response.json();

Making Requests

The Pitchey API is a RESTful API that uses standard HTTP methods and returns JSON responses. All requests should be made over HTTPS.

Base URLs

  • Production: https://pitchey-api-prod.ndlovucavelle.workers.dev
  • Local Development: http://localhost:8001

HTTP Methods

  • GET - Retrieve data
  • POST - Create new resources
  • PUT - Update existing resources
  • DELETE - Delete resources
  • PATCH - Partial updates

Content Types

The API accepts and returns JSON data. Always set the Content-Type header:

// Standard headers for API requests
const headers = {
  'Content-Type': 'application/json',
  'Authorization': `Bearer ${token}`
};

Error Handling

The API uses standard HTTP status codes to indicate success or failure:

Status Code Description Common Causes
200 OK Request successful
201 Created Resource created successfully
400 Bad Request Invalid request data or validation errors
401 Unauthorized Missing or invalid authentication token
403 Forbidden Insufficient permissions for the requested action
404 Not Found Requested resource doesn't exist
429 Rate Limited Too many requests - see rate limiting
500 Internal Server Error Server-side error occurred

Example Error Handling

async function apiRequest(url, options = {}) {
  try {
    const response = await fetch(url, {
      ...options,
      headers: {
        'Content-Type': 'application/json',
        'Authorization': `Bearer ${token}`,
        ...options.headers
      }
    });

    if (!response.ok) {
      const errorData = await response.json();
      throw new Error(`API Error (${response.status}): ${errorData.error}`);
    }

    return await response.json();
  } catch (error) {
    console.error('API request failed:', error);
    throw error;
  }
}

Rate Limiting

To ensure fair usage and maintain service quality, the API implements rate limiting:

Endpoint Type Rate Limit Window
Standard API calls 100 requests Per minute
File uploads 10 requests Per minute
Authentication 20 requests Per minute
WebSocket connections 5 connections Concurrent

Pagination

List endpoints support pagination using query parameters:

// Get the second page of pitches with 10 items per page
const response = await fetch('/api/pitches/public?page=2&limit=10');
const data = await response.json();

console.log(data.pitches);  // Array of pitch objects
console.log(data.meta);     // Pagination metadata

Pagination Response

{
  "pitches": [...],
  "meta": {
    "page": 2,
    "limit": 10,
    "total": 150,
    "totalPages": 15,
    "hasNext": true,
    "hasPrevious": true
  }
}

Filtering & Search

Many endpoints support filtering and search capabilities:

Basic Filtering

// Filter pitches by genre and format
const response = await fetch('/api/pitches/public?genre=sci-fi&format=feature');

// Search pitches
const searchResponse = await fetch('/api/pitches/search?q=space+thriller');

// Advanced search with multiple filters
const advancedResponse = await fetch('/api/search/advanced?' + new URLSearchParams({
  query: 'thriller',
  genres: ['thriller', 'action'],
  budgetRanges: ['1M-5M', '5M-20M'],
  seekingInvestment: true
}));

Webhooks

Webhooks allow you to receive real-time notifications when certain events occur in the Pitchey platform. Configure webhook endpoints in your developer dashboard.

Supported Events

  • pitch.created - New pitch was created
  • pitch.updated - Pitch was updated
  • nda.requested - NDA access was requested
  • nda.signed - NDA was signed
  • message.received - New message was received
  • investment.tracked - Investment was tracked

Webhook Payload Example

{
  "event": "pitch.created",
  "timestamp": "2024-01-01T12:00:00Z",
